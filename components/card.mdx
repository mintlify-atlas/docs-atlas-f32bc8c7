# Card

The Card component provides a versatile container for organizing related information. Built with a compositional API, it includes subcomponents for headers, titles, descriptions, content, actions, and footers.

## Installation

The Card component is available as part of the `@conty/ui` package:

```bash
npm install @conty/ui
```

## Usage

### Basic Card

```tsx
import { Card, CardHeader, CardTitle, CardContent } from "@conty/ui"

export default function Example() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Card Title</CardTitle>
      </CardHeader>
      <CardContent>
        <p>Card content goes here.</p>
      </CardContent>
    </Card>
  )
}
```

### Complete Card Structure

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
  CardAction,
  Button
} from "@conty/ui"

export default function Example() {
  return (
    <Card className="w-[360px]">
      <CardHeader>
        <CardTitle>Pro Plan</CardTitle>
        <CardDescription>
          Ideal for teams that need collaboration.
        </CardDescription>
        <CardAction>
          <Button variant="ghost" size="icon">
            <IconDotsVertical />
          </Button>
        </CardAction>
      </CardHeader>
      <CardContent>
        <p className="text-sm text-muted-foreground">
          Includes components, tokens, and priority support.
        </p>
      </CardContent>
      <CardFooter>
        <Button size="sm">Select plan</Button>
      </CardFooter>
    </Card>
  )
}
```

```tsx

```

## API Reference

### Card

The root container component.

<ParamField path="className" type="string">
  Additional CSS classes to apply to the card container.
</ParamField>

All standard HTML div attributes are supported.

### CardHeader

Container for the card's header section, including title, description, and optional action.

<ParamField path="className" type="string">
  Additional CSS classes to apply to the header.
</ParamField>

<Note>
  CardHeader uses a grid layout that automatically positions CardAction in the top-right corner when present.
</Note>

### CardTitle

Displays the main heading of the card.

<ParamField path="className" type="string">
  Additional CSS classes to apply to the title.
</ParamField>

### CardDescription

Displays supporting text below the title.

<ParamField path="className" type="string">
  Additional CSS classes to apply to the description.
</ParamField>

### CardAction

Container for header actions, positioned in the top-right corner of the card header.

<ParamField path="className" type="string">
  Additional CSS classes to apply to the action container.
</ParamField>

<Note>
  CardAction must be placed inside CardHeader to position correctly. It uses CSS Grid positioning to align to the top-right.
</Note>

### CardContent

The main content area of the card.

<ParamField path="className" type="string">
  Additional CSS classes to apply to the content area.
</ParamField>

### CardFooter

Container for footer content, typically actions or metadata.

<ParamField path="className" type="string">
  Additional CSS classes to apply to the footer.
</ParamField>

## Accessibility

### Semantic Structure

- Card components use semantic `<div>` elements
- Consider wrapping cards in `<article>` or `<section>` for better semantics:

```tsx
<article>
  <Card>
    <CardHeader>
      <CardTitle>Article Title</CardTitle>
    </CardHeader>
    <CardContent>
      {/* content */}
    </CardContent>
  </Card>
</article>
```

### Heading Hierarchy

CardTitle doesn't enforce heading level. Use appropriate heading elements:

```tsx
<CardHeader>
  <CardTitle asChild>
    <h2>Proper Heading</h2>
  </CardTitle>
</CardHeader>
```

### Interactive Cards

If the entire card is clickable, use proper interactive elements:

```tsx
<a href="/details" className="block no-underline">
  <Card className="hover:border-primary transition-colors">
    {/* card content */}
  </Card>
</a>
```

### Focus Management

Ensure interactive elements within cards (buttons, links) are keyboard accessible and have visible focus states.

## Best Practices

1. **Consistent spacing**: Use the default padding classes unless you have a specific layout need
2. **Header actions**: Place contextual actions in CardAction for consistent positioning
3. **Dividers**: Use `border-b` on CardHeader and `border-t` on CardFooter for visual separation
4. **Width constraints**: Cards work best with explicit width constraints (e.g., `w-[360px]`)
5. **Composition over props**: Build complex cards by composing subcomponents rather than passing complex prop configurations
6. **Semantic HTML**: Wrap CardTitle content in appropriate heading elements for accessibility
7. **Responsive layouts**: Use grid or flex layouts to arrange multiple cards
