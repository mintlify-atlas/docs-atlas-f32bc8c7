# Button

The Button component is a foundational interactive element built with Radix UI's Slot primitive, supporting multiple variants, sizes, and the ability to render as any element.

## Installation

The Button component is available as part of the `@conty/ui` package:

```bash
npm install @conty/ui
```

## Usage

### Basic Button

```tsx
import { Button } from "@conty/ui"

export default function Example() {
  return <Button>Click me</Button>
}
```

```tsx

export default function Example() {
  return (
    <>
      <Button>
        <IconDownload />
        Download
      </Button>
      
      <Button size="icon">
        <IconDownload />
      </Button>
    </>
  )

```

### As Child (Polymorphic)

Use the `asChild` prop to render the button as a different element while maintaining button styling:

```tsx
import { Button } from "@conty/ui"

export default function Example() {
  return (
    <Button asChild>
      <a href="/dashboard">Go to Dashboard</a>
    </Button>
  )
}
```

## API Reference

### Button Props

The Button component extends all standard HTML button attributes and adds the following:

<ParamField path="variant" type="string" default="default">
  The visual style variant of the button.
  
  Options:
  - `default` - Primary button with filled background
  - `destructive` - For destructive actions (delete, remove, etc.)
  - `outline` - Outlined button with transparent background
  - `secondary` - Secondary emphasis button
  - `ghost` - Minimal button without border
  - `link` - Text button with underline on hover
</ParamField>

<ParamField path="size" type="string" default="default">
  The size of the button.
  
  Options:
  - `default` - Standard button height (h-9)
  - `sm` - Small button (h-8)
  - `lg` - Large button (h-10)
  - `icon` - Square icon button (size-9)
  - `icon-sm` - Small square icon button (size-8)
  - `icon-lg` - Large square icon button (size-10)
</ParamField>

<ParamField path="asChild" type="boolean" default="false">
  When true, the button will render as a Slot component, merging props with its child element. This is useful for rendering buttons as links or other elements.
</ParamField>

<ParamField path="disabled" type="boolean" default="false">
  When true, disables the button and prevents user interaction.
</ParamField>

<ParamField path="className" type="string">
  Additional CSS classes to apply to the button.
</ParamField>

### TypeScript Definition

```tsx
import { VariantProps } from "class-variance-authority"

type ButtonProps = React.ComponentProps<"button"> &
  VariantProps<typeof buttonVariants> & {
    asChild?: boolean
  }
```

## Accessibility

### Keyboard Support

- `Enter` or `Space` activates the button
- `Tab` moves focus to the next focusable element
- `Shift+Tab` moves focus to the previous focusable element

### Focus Management

The button includes visible focus indicators:

```tsx
focus-visible:border-ring focus-visible:ring-ring/50 focus-visible:ring-[3px]
```

### ARIA Attributes

- Use `aria-busy="true"` for loading states
- Use `aria-invalid="true"` for error states (displays red focus ring)
- The `disabled` prop automatically adds `aria-disabled`

### Screen Reader Support

- Uses semantic `<button>` element by default
- When using `asChild` with links, ensure proper `role` attributes if needed
- For icon-only buttons, include `aria-label` or `title`:

```tsx
<Button size="icon" aria-label="Download file">
  <IconDownload />
</Button>
```

<Note>
  When using the `asChild` prop with a link, the component renders as an anchor element with button styling. Ensure you're using the appropriate element for your use case to maintain proper semantics.
</Note>

## Best Practices

1. **Use semantic variants**: Choose destructive for dangerous actions, primary for main CTAs, and secondary for alternative actions
2. **Icon buttons need labels**: Always provide `aria-label` for icon-only buttons
3. **Loading states**: Use `disabled` and `aria-busy` together for loading states
4. **Link semantics**: Use `asChild` with anchor tags for navigation, not `onClick` handlers
5. **Consistent sizing**: Use the same size buttons in button groups for visual harmony
